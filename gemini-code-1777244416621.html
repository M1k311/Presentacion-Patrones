¿Qué son los Patrones Estructurales? [cite: 5]

Son el lenguaje común que usan los ingenieros de software en todo el mundo[cite: 6].

Describen cómo ensamblar objetos y clases en estructuras más grandes, al tiempo que se mantienen esas estructuras flexibles y eficientes[cite: 7].

En resumen: Son soluciones probadas a problemas recurrentes de composición de código[cite: 8].

1. Decorator (Wrapper) [cite: 10, 26]

Añade comportamientos a objetos envolviéndolos en capas, sin tocar su código original. Como poner ropa encima de ropa[cite: 11].

Analogía: Ponerse ropa. Un suéter extiende tu comportamiento "tienes frío", una chaqueta añade otra capa. Puedes quitarte capas sin afectar el resto[cite: 35, 36, 37, 38, 39].

Industria: Uber (Añade tokens, red, seguridad al viaje) [cite: 41, 42]

class CafeBase:
    def costo(self): return 20.0  #[cite: 31]

class DecoradorLeche:
    def __init__(self, cafe):
        self.cafe = cafe
    def costo(self):
        return self.cafe.costo() + 5.0

mi_cafe = CafeBase()
cafe_con_leche = DecoradorLeche(mi_cafe) #[cite: 31]

2. Facade [cite: 17, 43]

Proporciona una interfaz simplificada a una librería, un framework o conjunto complejo de clases[cite: 44, 45, 46, 47]. Oculta la complejidad[cite: 47].

Analogía: Como un call center. No necesitas conocer a los 15 departamentos; hablas con una sola persona que coordina todo[cite: 48, 49, 50].

Industria: Mercado Pago [cite: 101] Stripe [cite: 100]

class FachadaTienda: #[cite: 72]
    def __init__(self):
        self.inv = Inventario() #[cite: 78]
        self.banco = Banco() #[cite: 80]
        self.envio = Envio() #[cite: 81]

    def comprar_articulo(self): #[cite: 88]
        if self.inv.hay_stock(): #[cite: 89]
            self.banco.procesar_pago() #[cite: 90]
            self.envio.enviar_paquete() #[cite: 91]
            
mi_tienda = FachadaTienda() #[cite: 96]
mi_tienda.comprar_articulo() #[cite: 98]

3. Flyweight [cite: 13, 102]

Optimiza el consumo de memoria RAM compartiendo datos comunes entre múltiples objetos[cite: 104, 105, 106, 107, 108]. Se mantiene una única copia compartida[cite: 116].

Analogía: Los sellos. No fabricas mil moldes para marcar mil hojas; usas un solo molde y cambias el papel[cite: 124, 125].


class Fabrica: #[cite: 118]
    _sello = None #[cite: 131]
    
    @classmethod #[cite: 132]
    def obtener_sello(cls): #[cite: 133]
        if not cls._sello: cls._sello = Sello() #[cite: 134]
        return cls._sello #[cite: 134]

sello = Fabrica.obtener_sello() #[cite: 136]
for i in range(1000): #[cite: 137]
    print(f"Hoja {i} sellada") #[cite: 138]

4. Proxy [cite: 21, 140]

Proporciona un sustituto, marcador de posición o representante para controlar el acceso a otro objeto. Puede retrasar la carga (Lazy loading)[cite: 141, 142, 25].

Analogía: Un Asistente. Recibe peticiones y solo te pasa con el jefe si tienes cita o es urgente[cite: 143, 144, 145].

Industria: Netflix [cite: 179] AWS [cite: 180]

class AsistenteProxy: #[cite: 156]
    def __init__(self): #[cite: 170]
        self.directivo = None #[cite: 171]
        
    def peticion(self, urgente): #[cite: 172]
        if urgente: #[cite: 173]
            if not self.directivo: self.directivo = DirectivoReal() #[cite: 175]
            self.directivo.atender() #[cite: 175]

proxy = AsistenteProxy() #[cite: 168]
proxy.peticion(urgente=True) #[cite: 177]
