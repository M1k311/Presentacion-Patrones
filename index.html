Composición y Estructura

Los patrones estructurales nos enseñan a ensamblar objetos y clases en estructuras más grandes, manteniendo el sistema flexible y eficiente.

El Problema

A medida que un sistema crece, las relaciones entre clases se vuelven rígidas. La herencia tradicional crea jerarquías difíciles de mantener.

La Solución

Utilizar la composición para envolver objetos o crear fachadas que simplifiquen la interacción entre componentes.

1. Decorator (Wrapper)

Concepto

Añade responsabilidades adicionales a un objeto de forma dinámica. Es la alternativa ideal para evitar la "explosión de clases".

Analogía: Como ponerse ropa. Un suéter extiende tu comportamiento si tienes frío, una chaqueta añade otra capa. Puedes quitar capas sin afectarte a ti mismo.
Industria Uber Java I/O 
class CafeBase:
    def costo(self): return 20.0

class DecoradorLeche:
    def __init__(self, cafe):
        self.cafe = cafe
    def costo(self):
        return self.cafe.costo() + 5.0

# Cliente envuelve el objeto base
mi_pedido = DecoradorLeche(CafeBase())

2. Facade (Fachada)

Concepto

Proporciona una interfaz unificada y simple para un subsistema complejo de clases, ocultando la lógica interna al cliente.

Analogía: Como un operador de Call Center. No necesitas conocer a los 15 departamentos internos; hablas con una persona que coordina todo.
Industria Stripe Mercado Pago 
class FachadaTienda:
    def __init__(self):
        self.inv = Inventario()
        self.banco = Banco()
        self.envio = Envio()

    def comprar(self):
        if self.inv.hay_stock():
            self.banco.pago()
            self.envio.mandar()

3. Flyweight (Peso Ligero)

Concepto

Optimiza el uso de RAM compartiendo datos comunes (estado intrínseco) entre múltiples objetos similares en lugar de duplicarlos.

Analogía: Los sellos. No fabricas mil moldes para marcar mil hojas; usas un solo molde y solo cambias el papel para ahorrar material y espacio.
Industria Google Maps PDF Readers 
class Fabrica:
    _sello = None
    @classmethod
    def obtener_sello(cls):
        if not cls._sello: 
            cls._sello = SelloPesado()
        return cls._sello

# Miles de marcas, un solo objeto en RAM
sello = Fabrica.obtener_sello()

4. Proxy (Intermediario)

Concepto

Actúa como sustituto de un objeto para controlar el acceso, permitir la carga diferida (Lazy Loading) o añadir seguridad.

Analogía: Un Asistente Personal. Recibe peticiones y solo te pasa con el jefe si es urgente o tienes cita, evitando saturar el recurso real.
Industria Netflix Nginx / AWS 
class ProxySeguridad:
    def __init__(self):
        self.real = RecursoPesado()
    
    def peticion(self, usuario):
        if usuario == "Admin":
            self.real.acceder()
        else:
            print("Acceso Denegado por el Proxy")
