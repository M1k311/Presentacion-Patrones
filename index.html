Composición y Estructura

Los patrones estructurales nos enseñan a ensamblar objetos y clases en estructuras más grandes, manteniendo el sistema flexible y eficiente.

Problema Recurrente

Cuando un sistema crece, las relaciones entre clases se vuelven rígidas. La herencia tradicional crea "explosión de clases" difíciles de mantener.

Solución de Diseño

Utilizar la composición para envolver objetos o crear fachadas que oculten la complejidad interna del sistema.

1. Decorator (Wrapper)

¿Qué es?

Añade responsabilidades adicionales a un objeto de forma dinámica sin modificar su código base. Es una alternativa flexible a la herencia.

Analogía: Como ponerse ropa. Un suéter extiende tu comportamiento si tienes frío, una chaqueta añade otra capa. Puedes quitar capas sin afectarte a ti mismo.
Industria Uber Java I/O Streams 
class CafeBase:
    def costo(self): return 20.0

class DecoradorLeche:
    def __init__(self, cafe):
        self.cafe = cafe
    def costo(self):
        return self.cafe.costo() + 5.0

# Uso dinámico
mi_pedido = DecoradorLeche(CafeBase())

2. Facade (Fachada)

¿Qué es?

Proporciona una interfaz simplificada a una biblioteca, un framework o cualquier conjunto complejo de clases de un subsistema.

Analogía: Como un operador de Call Center. No necesitas conocer a los 15 departamentos internos; hablas con una persona que coordina todo por ti.
Industria Stripe Mercado Pago 
class FachadaTienda:
    def __init__(self):
        self.inv = Inventario()
        self.banco = Banco()
        self.envio = Envio()

    def comprar(self):
        if self.inv.hay_stock():
            self.banco.pago()
            self.envio.mandar()

3. Flyweight (Peso Ligero)

¿Qué es?

Permite mantener más objetos en la memoria RAM compartiendo las partes comunes del estado entre varios objetos en lugar de mantener todos los datos en cada uno.

Analogía: Los sellos. No fabricas mil moldes para marcar mil hojas; usas un solo molde y solo cambias de lugar el papel para ahorrar material.
Industria Google Maps Adobe Acrobat 
class Fabrica:
    _sello = None
    @classmethod
    def obtener_sello(cls):
        if not cls._sello: 
            cls._sello = SelloPesado()
        return cls._sello

# Mil hojas, una sola imagen en RAM
sello = Fabrica.obtener_sello()

4. Proxy (Intermediario)

¿Qué es?

Proporciona un sustituto o marcador de posición para otro objeto para controlar el acceso a él, permitiendo hacer algo antes o después de que la solicitud llegue al objeto real.

Analogía: Un Asistente Personal. Recibe tus peticiones y solo te pasa con el jefe si es urgente o tienes cita, evitando que él se sature.
Industria Netflix Nginx / AWS 
class ProxySeguridad:
    def __init__(self):
        self.real = RecursoPesado()
    
    def peticion(self, usuario):
        if usuario == "Admin":
            self.real.acceder()
        else:
            print("Acceso Denegado")
