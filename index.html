¿Qué son los Patrones Estructurales?

En la ingeniería de software, los patrones estructurales se enfocan en cómo las clases y objetos se componen para formar estructuras más grandes y complejas, garantizando que el sistema sea flexible y eficiente.

  • Herencia vs Composición: En lugar de depender exclusivamente de la herencia (que es estática y rígida), estos patrones favorecen la composición de objetos en tiempo de ejecución.
  • Objetivo: Simplificar el diseño identificando una forma estandarizada de organizar las relaciones entre las entidades de código.

1. Decorator (Envoltorio / Wrapper)

Permite añadir responsabilidades y comportamientos a un objeto de forma dinámica. Es la alternativa perfecta a la herencia múltiple para evitar la "explosión de clases" (crear subclases para cada combinación posible de características).

Principio SOLID aplicado: Open/Closed Principle (Abierto a la extensión, cerrado a la modificación).


# En lugar de crear CafeConLecheYAzucar, CafeConLeche, CafeConAzucar...
class CafeBase:
    def costo(self): return 20.0

class DecoradorLeche:
    def __init__(self, cafe):
        self.cafe = cafe # Recibe el objeto original (Composición)
    def costo(self):
        return self.cafe.costo() + 5.0 # Añade su propio comportamiento

mi_cafe = CafeBase()
cafe_con_leche = DecoradorLeche(mi_cafe) # El cliente decide qué capas añadir

2. Facade (Fachada)

Proporciona una interfaz unificada de alto nivel para un subsistema complejo. Su objetivo es desacoplar al cliente del subsistema, reduciendo la curva de aprendizaje y las dependencias.

Casos de uso en la Industria: APIs de pasarelas de pago (Stripe, Mercado Pago). Detrás de un simple `procesarPago()`, hay sistemas de prevención de fraude, validación de tarjetas y conexión bancaria.


class FachadaTienda:
    def __init__(self):
        # Inicializa todos los subsistemas complejos
        self.inventario = Inventario()
        self.banco = Banco()
        self.logistica = Envio()

    def comprar_articulo(self):
        # Oculta la complejidad de la orquestación
        if self.inventario.hay_stock():
            self.banco.procesar_pago()
            self.logistica.enviar_paquete()

3. Flyweight (Peso Ligero)

Diseñado para sistemas que manejan miles o millones de objetos, optimizando drásticamente el uso de memoria RAM al dividir el estado en dos:

  • Estado Intrínseco: Inmutable y compartido entre todos los objetos (ej. la textura gráfica de un árbol en un videojuego).
  • Estado Extrínseco: Varía y se pasa como contexto (ej. las coordenadas X, Y de cada árbol).

class Fabrica:
    _recurso_pesado = None # Memoria compartida (Estado Intrínseco)
    
    @classmethod
    def obtener_recurso(cls):
        if not cls._recurso_pesado: 
            cls._recurso_pesado = CargarArchivoGigante()
        return cls._recurso_pesado

# 100,000 llamadas, pero solo 1 instancia en RAM
for i in range(100000):
    Fabrica.obtener_recurso().dibujar(x=i, y=i) # x, y es Estado Extrínseco

4. Proxy (Intermediario)

Proporciona un sustituto o representante para controlar el acceso a otro objeto. Existen varios tipos:

  • Proxy Virtual: Retrasa la creación de un objeto pesado hasta que sea estrictamente necesario (Lazy Loading).
  • Proxy de Protección: Verifica credenciales antes de redirigir la petición al objeto real.

class AsistenteProxy:
    def __init__(self):
        self.directivo = None # No se instancia al inicio
        
    def peticion(self, es_urgente):
        if es_urgente:
            # Lazy Loading: Solo se carga si se cumple la condición
            if not self.directivo: 
                self.directivo = DirectivoReal()
            self.directivo.atender()
        else:
            print("Acceso denegado o pospuesto por el Proxy.")
